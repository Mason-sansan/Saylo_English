# GStack Plan CEO Review (Cursor)

## Purpose
Make the plan sharper: fewer promises, clearer wedge, stronger differentiation.

## Default stance
Assume the plan is too broad. Reduce until it becomes shippable in 1–3 days.

## What to evaluate
- **Promise**: what the user gets in one sentence
- **Wedge**: the smallest credible product that delivers the promise
- **Non-goals**: what is explicitly out
- **Differentiation**: why this is meaningfully better than generic alternatives
- **Failure mode**: how this dies in the real world

## Output format
- **Verdict**: HoldScope / Reduce / Expand (pick one)
- **Reframe**: 1 sentence
- **Cuts**: 3 bullets (what to drop)
- **Keeps**: 3 bullets (what stays)
- **Next plan**: 5–10 bullets, ordered, testable
