# GStack QA-Only (Cursor)

## What to do
- Enumerate the critical user paths.
- For each path, list steps and what to verify.
- Record issues as crisp bugs (not advice).

## Bug format
- **Title**
- **Severity**: blocker / high / medium / low
- **Steps**
- **Expected**
- **Actual**
- **Notes**: likely cause or related areas (optional)
